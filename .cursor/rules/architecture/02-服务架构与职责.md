## 服务层次架构

### 整体分层

```
┌─────────────────────────────────────────────────────────────┐
│                        前端层                                │
│                    (Vue 3 Components)                       │
└────────────────────────────┬────────────────────────────────┘
                             │ HTTP API
┌────────────────────────────▼────────────────────────────────┐
│                      Controller 层                           │
│  (API 接口定义、请求响应处理、参数验证)                      │
└────────────────────────────┬────────────────────────────────┘
                             │ 调用
┌────────────────────────────▼────────────────────────────────┐
│                       Service 层                             │
│  (业务逻辑、协调、事务管理、规则验证)                        │
└────────────────────────────┬────────────────────────────────┘
                             │ 协调
         ┌───────────────────┼───────────────────┐
         │                   │                   │
┌────────▼────────┐  ┌──────▼───────┐  ┌───────▼──────┐
│  Executor 层    │  │  Agent 层    │  │  Tool 层     │
│ (任务执行调度)  │  │ (AI 决策)    │  │ (功能实现)   │
└─────────────────┘  └──────────────┘  └──────────────┘
         │                   │                   │
         └───────────────────┼───────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────┐
│                      Repository 层                           │
│               (数据访问、持久化)                             │
└─────────────────────────────────────────────────────────────┘
```

### 职责划分原则

| 层级 | 核心职责 | 不应包含 |
|------|----------|----------|
| Controller | API 定义、参数验证、响应封装 | 业务逻辑 |
| Service | 业务逻辑、协调、事务 | 数据访问细节 |
| Executor | 任务执行、调度控制 | 业务决策 |
| Agent | AI 决策、工具选择 | 具体功能实现 |
| Tool | 具体功能实现 | 业务逻辑 |
| Repository | 数据访问 | 业务逻辑 |

### Controller 层职责规范

```java
// ✅ 正确的 Controller 实现
@RestController
@RequestMapping("/api/executor")
public class LynxeController {

    private final PlanningCoordinator planningCoordinator;

    /**
     * 异步执行计划
     */
    @PostMapping("/executeByToolNameAsync")
    public ResponseEntity<AsyncExecutionResponse> executeAsync(
            @RequestBody @Valid ExecuteByToolNameRequest request) {

        // 1. 参数验证（通过 @Valid）
        // 2. 调用 Service 层
        String planId = planningCoordinator.executeAsync(request);

        // 3. 构建响应
        AsyncExecutionResponse response = new AsyncExecutionResponse();
        response.setPlanId(planId);
        response.setStatus("processing");

        return ResponseEntity.ok(response);
    }
}

// ❌ 错误的 Controller 实现
@RestController
@RequestMapping("/api/executor")
public class LynxeController {

    /**
     * 直接在 Controller 中实现业务逻辑
     */
    @PostMapping("/executeByToolNameAsync")
    public ResponseEntity<AsyncExecutionResponse> executeAsync(
            @RequestBody ExecuteByToolNameRequest request) {

        // 不应包含业务逻辑
        if (request.getToolName() == null) {
            throw new IllegalArgumentException("Tool name is required");
        }

        // 不应直接访问数据库
        PlanTemplateEntity entity = repository.findByToolName(request.getToolName());

        // 不应直接创建 Executor
        Executor executor = new Executor();
        String planId = executor.execute(entity);

        return ResponseEntity.ok(response);
    }
}
```

### Service 层职责规范

```java
// ✅ 正确的 Service 实现
@Service
public class PlanTemplateService {

    private final PlanTemplateVersionRepository repository;
    private static final Logger logger = LoggerFactory.getLogger(PlanTemplateService.class);

    @Transactional
    public Integer saveToVersionHistory(PlanTemplateConfigVO template) {
        // 1. 业务验证
        if (template == null) {
            throw new ParameterValidationException("template", "Template cannot be null");
        }

        // 2. 业务逻辑
        boolean isSame = isContentSameAsLatestVersion(
            template.getPlanTemplateId(),
            template.getPlanJson()
        );

        if (isSame) {
            logger.info("Content is same as latest version, skipping save");
            return getLatestVersionIndex(template.getPlanTemplateId());
        }

        // 3. 数据持久化
        PlanTemplateVersion entity = convertToEntity(template);
        entity.setVersionIndex(getNextVersionIndex(template.getPlanTemplateId()));
        PlanTemplateVersion saved = repository.save(entity);

        // 4. 返回结果
        return saved.getVersionIndex();
    }
}
```

### Executor 层职责规范

```java
// ✅ 正确的 Executor 实现
@Component
public class DynamicToolPlanExecutor extends AbstractPlanExecutor {

    private final BaseAgentFactory agentFactory;
    private final PlanExecutionRecorder recorder;

    @Override
    public String execute(AbstractExecutionPlan plan) {
        String planId = generatePlanId();

        try {
            // 1. 开始记录
            recorder.startRecording(planId, plan.getContext());

            // 2. 创建并执行 Agent
            BaseAgent agent = agentFactory.createAgent(plan.getAgentType());
            AgentExecutionResult result = agent.execute(plan.getContext());

            // 3. 记录结果
            recorder.addAgentExecution(planId, result.toRecord());

            return planId;
        } catch (Exception e) {
            // 4. 错误处理
            recorder.updateStatus(planId, ExecutionStatus.FAILED);
            throw new PlanExecutionException(planId, "Execution failed", e);
        }
    }
}
```

```java
// ✅ 正确的 Agent 实现
public abstract class BaseAgent {

    protected final LlmService llmService;
    protected final List<Tool> availableTools;

    /**
     * 执行 Agent 的模板方法
     */
    public final AgentExecutionResult execute(ExecutionContext context) {
        // 1. 准备执行
        prepareExecution(context);

        // 2. 构建 prompt
        List<Message> messages = buildMessages(context);

        // 3. 调用 LLM
        ChatResponse response = llmService.chat(messages);

        // 4. 处理工具调用
        if (response.hasToolCalls()) {
            return processToolCalls(response.getToolCalls(), context);
        }

        // 5. 返回结果
        return buildResult(response);
    }

    /**
     * 子类实现具体的消息构建逻辑
     */
```

```java
public abstract class AbstractBaseTool {

    public final ToolResult execute(ToolInput input) {
```

## 服务调用关系

### 典型调用链路

#### 执行计划流程

```
LynxeController.executeByToolNameAsync()
    ↓
PlanningCoordinator.executeAsync()
    ↓
PlanningFactory.createPlan()
    ↓
PlanExecutorFactory.createExecutor()
    ↓
DynamicToolPlanExecutor.execute()
    ↓
BaseAgent.execute()
    ↓
Tool.execute()
    ↓
PlanExecutionRecorder.addAgentExecution()
```

#### 保存模板流程

```
PlanTemplateController.save()
    ↓
PlanTemplateService.saveToVersionHistory()
    ↓
PlanTemplateVersionRepository.save()
```

### 跨层调用规则

| 调用方向 | 允许 | 说明 |
|----------|------|------|
| Controller → Service | ✅ | 通过接口调用 |
| Controller → Repository | ❌ | 必须通过 Service |
| Service → Executor | ✅ | 协调执行 |
| Service → Repository | ✅ | 数据访问 |
| Service → Service | ✅ | 服务协作 |
| Executor → Agent | ✅ | 执行代理 |
| Executor → Tool | ✅ | 执行工具 |
| Agent → Tool | ✅ | 使用工具 |
| Tool → Repository | ✅ | 需要数据时 |
| Repository → Service | ❌ | 单向依赖 |
