## 数据模型概述

### 模型分类

Lynxe 使用分层的数据模型设计：

```
┌─────────────────────────────────────────────────────────────┐
│                    数据模型层次                             │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  PO (Persistent Object)                               │  │
│  │  - 对应数据库表                                        │  │
│  │  - JPA 实体                                           │  │
│  │  - Repository 直接操作                                 │  │
│  └───────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  VO (Value Object)                                    │  │
│  │  - 业务数据传输                                        │  │
│  │  - 跨层通信                                            │  │
│  │  - 与 PO 相互转换                                      │  │
│  └───────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  DTO (Data Transfer Object)                           │  │
│  │  - API 请求响应                                        │  │
│  │  - 前后端交互                                          │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 命名约定

| 类型 | 后缀 | 示例 |
|------|------|------|
| 持久化对象 | Entity | `PlanExecutionRecordEntity` |
| 值对象 | 无后缀 | `PlanExecutionRecord` |
| 数据传输对象 | DTO、Request、Response | `PlanExecutionRecordDTO` |

## 核心实体模型

### 1. PlanExecutionRecordEntity

**位置**: `recorder/entity/po/PlanExecutionRecordEntity.java`

**用途**: 记录计划执行的完整信息

**表结构**:
```sql
CREATE TABLE plan_execution_records (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    current_plan_id VARCHAR(255) NOT NULL UNIQUE,
    root_plan_id VARCHAR(255),
    parent_plan_id VARCHAR(255),
    title VARCHAR(500),
    user_request LONGTEXT,
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    current_step_index INT,
    completed BOOLEAN DEFAULT FALSE,
    summary TEXT,
    status VARCHAR(50),
    error_message TEXT,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**实体定义**:
```java
@Entity
@Table(name = "plan_execution_records")
public class PlanExecutionRecordEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "current_plan_id", nullable = false, unique = true)
    private String currentPlanId;

    @Column(name = "root_plan_id")
    private String rootPlanId;

    @Column(name = "parent_plan_id")
    private String parentPlanId;

    @Column(name = "title")
    private String title;

    @Column(name = "user_request", columnDefinition = "LONGTEXT")
    private String userRequest;

    @Column(name = "start_time")
    private LocalDateTime startTime;

    @Column(name = "end_time")
    private LocalDateTime endTime;

    @Column(name = "current_step_index")
    private Integer currentStepIndex;

    @Column(name = "completed")
    private Boolean completed = false;

    @Column(name = "summary")
    private String summary;

    @Column(name = "status")
    private String status;

    @Column(name = "error_message")
    private String errorMessage;

    @OneToMany(cascade = CascadeType.ALL, mappedBy = "planExecution", fetch = FetchType.LAZY)
    private List<AgentExecutionRecordEntity> agentExecutionSequence;

    // Getters and Setters
}
```

## 值对象模型

### 1. PlanExecutionRecord (VO)

**位置**: `recorder/entity/vo/PlanExecutionRecord.java`

**用途**: 业务层使用的计划执行记录

**定义**:
```java
public class PlanExecutionRecord {

    private String currentPlanId;
    private String rootPlanId;
    private String parentPlanId;
    private String title;
    private String userRequest;
    private LocalDateTime startTime;
    private LocalDateTime endTime;
    private Integer currentStepIndex;
    private Boolean completed;
    private String summary;
    private String status;
    private String errorMessage;
    private List<AgentExecutionRecord> agentExecutionSequence;

    // 业务方法
    public boolean isRunning() {
        return "RUNNING".equals(status);
    }

    public boolean isCompleted() {
        return Boolean.TRUE.equals(completed);
    }

    public AgentExecutionRecord getCurrentStep() {
        if (agentExecutionSequence == null ||
            agentExecutionSequence.isEmpty()) {
            return null;
        }
        if (currentStepIndex == null) {
            currentStepIndex = 0;
        }
        return agentExecutionSequence.get(currentStepIndex);
    }
}
```

### 2. AgentExecutionRecord (VO)

**位置**: `recorder/entity/vo/AgentExecutionRecord.java`

**定义**:
```java
public class AgentExecutionRecord {

    private String stepId;
    private String agentName;
    private String agentRequest;
    private String agentDescription;
    private String result;
    private String errorMessage;
    private String status;
    private LocalDateTime startTime;
    private LocalDateTime endTime;
    private List<ThinkActRecord> thinkActRecords;

    // 业务方法
    public boolean isSuccessful() {
        return "FINISHED".equals(status) &&
               (errorMessage == null || errorMessage.isEmpty());
    }

    public Duration getDuration() {
        if (startTime == null || endTime == null) {
            return Duration.ZERO;
        }
        return Duration.between(startTime, endTime);
    }
}
```

## 数据库设计

### 数据库选择

Lynxe 支持多种数据库：

| 数据库 | 默认 | 适用场景 |
|--------|------|----------|
| H2 | ✅ | 开发、测试、小型部署 |
| MySQL | - | 生产环境 |
| PostgreSQL | - | 生产环境 |

### 配置切换

**H2 (默认)**:
```yaml
# application-h2.yml
spring:
  datasource:
    url: jdbc:h2:file:./h2-data/openlynxe_db
    driver-class-name: org.h2.Driver
    username: sa
    password:
  jpa:
    database-platform: org.hibernate.dialect.H2Dialect
```

**MySQL**:
```yaml
# application-mysql.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lynxe?useSSL=false&serverTimezone=Asia/Shanghai
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: lynxe
    password: your_password
  jpa:
    database-platform: org.hibernate.dialect.MySQLDialect
```

**PostgreSQL**:
```yaml
# application-postgres.yml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/lynxe
    driver-class-name: org.postgresql.Driver
    username: lynxe
    password: your_password
  jpa:
    database-platform: org.hibernate.dialect.PostgreSQLDialect
```

### 数据转换

#### PO 到 VO 转换

```java
// recorder/entity/vo/PlanExecutionRecord.java
public static PlanExecutionRecord fromEntity(PlanExecutionRecordEntity entity) {
    if (entity == null) {
        return null;
    }

    PlanExecutionRecord vo = new PlanExecutionRecord();
    vo.setCurrentPlanId(entity.getCurrentPlanId());
    vo.setRootPlanId(entity.getRootPlanId());
    vo.setParentPlanId(entity.getParentPlanId());
    vo.setTitle(entity.getTitle());
    vo.setUserRequest(entity.getUserRequest());
    vo.setStartTime(entity.getStartTime());
    vo.setEndTime(entity.getEndTime());
    vo.setCurrentStepIndex(entity.getCurrentStepIndex());
    vo.setCompleted(entity.getCompleted());
    vo.setSummary(entity.getSummary());
    vo.setStatus(entity.getStatus());
    vo.setErrorMessage(entity.getErrorMessage());

    // 转换关联的 Agent 执行记录
    if (entity.getAgentExecutionSequence() != null) {
        List<AgentExecutionRecord> agentRecords = entity.getAgentExecutionSequence().stream()
            .map(AgentExecutionRecord::fromEntity)
            .collect(Collectors.toList());
        vo.setAgentExecutionSequence(agentRecords);
    }

    return vo;
}
```

### API 响应映射

**后端响应**:
```java
// Controller 返回
@GetMapping("/details/{planId}")
public ResponseEntity<PlanExecutionRecord> getDetails(@PathVariable String planId) {
    PlanExecutionRecordEntity entity = repository.findByCurrentPlanId(planId)
        .orElseThrow(() -> new PlanNotFoundException(planId));
    PlanExecutionRecord vo = PlanExecutionRecord.fromEntity(entity);
    return ResponseEntity.ok(vo);
}
```

## 存储策略

### 文件存储

```
./
├── h2-data/                    # H2 数据库文件
│   └── openlynxe_db.mv.db
├── uploads/                    # 上传文件临时目录
│   └── upload-{timestamp}_{random}_{threadId}/
│       ├── file1.pdf
│       └── file2.txt
├── workspace/                  # 执行工作空间
│   └── {planId}/
│       ├── files/             # 同步的文件
│       ├── output/            # 执行输出
│       └── temp/              # 临时文件
└── logs/                      # 日志文件
    └── lynxe.log
```

### 清理策略

**上传文件清理**:
- 执行完成后自动清理
- 可配置保留时间
- 定期清理任务

**工作空间清理**:
- 执行完成后可选清理
- 失败执行强制清理
- 定期清理过期工作空间
