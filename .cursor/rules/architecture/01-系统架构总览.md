## 系统概述

### 系统定位

Lynxe 是一个基于 Spring AI Alibaba 构建的 AI Agent 管理系统，提供了：

1. **多代理协作**: 支持 Agent 之间的协作和任务分解
2. **Func-Agent 模式**: 精确控制执行细节
3. **MCP 集成**: 原生支持 Model Context Protocol
4. **HTTP 服务**: 提供 RESTful API 供外部系统调用
5. **文件处理**: 支持 PDF、Excel、Word 等多种文件格式

### 系统特点

```
┌─────────────────────────────────────────────────────────┐
│                    Lynxe 系统                            │
├─────────────────────────────────────────────────────────┤
│  ✅ 纯 Java 实现                                        │
│  ✅ 支持 LLM 多模型（OpenAI、DashScope 等）            │
│  ✅ 可扩展的工具系统                                    │
│  ✅ 计划模板版本管理                                    │
│  ✅ 执行过程可追踪                                      │
│  ✅ 多数据库支持（H2、MySQL、PostgreSQL）               │
│  ✅ 国际化支持（中英文）                                │
└─────────────────────────────────────────────────────────┘
```

### 架构分层说明

| 层级 | 职责 | 技术组件 |
|------|------|----------|
| **前端层** | 用户交互、数据展示 | Vue 3、TypeScript、Pinia、Ant Design Vue |
| **Controller 层** | API 接口、请求响应 | Spring Web、RestController |
| **Service 层** | 业务逻辑、协调 | Spring Service、业务组件 |
| **Agent 层** | AI 决策、规划 | BaseAgent、DynamicAgent |
| **Executor 层** | 任务执行、调度 | Executor、线程池 |
| **Tool 层** | 具体功能实现 | 内置工具、MCP 工具 |
| **LLM 层** | 大模型调用 | Spring AI、OpenAI API |
| **数据层** | 数据持久化 | JPA、Hibernate、数据库 |

### 配置管理方式

```java
// 方式 1: 配置文件
@Value("${lynxe.agent.maxSteps}")
private Integer maxSteps;

// 方式 2: 配置属性
@ConfigProperty(group = "lynxe", subGroup = "agent", key = "maxSteps")
private volatile Integer maxSteps;

// 方式 3: 运行时配置更新
configService.updateConfig("lynxe.agent.maxSteps", "200");
```
