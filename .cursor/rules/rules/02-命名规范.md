## 通用命名原则

### 核心原则
1. **语义化**: 名称应该清晰地表达其用途
2. **一致性**: 相同概念使用相同的命名
3. **简洁性**: 在保证清晰的前提下尽量简短
4. **英文优先**: 使用英文命名，避免拼音或中文

### 禁止事项
- ❌ 使用拼音命名
- ❌ 使用中文命名
- ❌ 使用无意义的缩写（除非是业界标准）
- ❌ 使用数字后缀区分（如 `user1`, `user2`）
- ❌ 使用保留字作为名称

## Java 命名规范

### 包命名 (Package Naming)

#### 规则
- 全小写字母
- 使用域名反转规则
- 包名使用名词
- 子包名描述功能模块

#### 示例
```java
// ✅ 正确
com.alibaba.cloud.ai.lynxe.agent
com.alibaba.cloud.ai.lynxe.planning.service
com.alibaba.cloud.ai.lynxe.runtime.controller

// ❌ 错误
Com.alibaba.cloud.AI.Lynxe.Agent
com.alibaba.cloud.ai.lynxe.Agent
com.alibaba.cloud.ai.lynxe.agentServices
```

### 类命名 (Class Naming)

#### 规则
- **PascalCase**（首字母大写）
- 使用名词
- 接口可以形容词结尾（如 `Service`, `Manager`, `Provider`）
- 异常类以 `Exception` 结尾
- 测试类以 `Test` 结尾

#### 类后缀约定

| 后缀 | 用途 | 示例 |
|------|------|------|
| `Service` | 业务逻辑服务类 | `PlanTemplateService`, `McpService` |
| `Controller` | 控制器类 | `LynxeController`, `PlanTemplateController` |
| `Repository` | 数据访问类 | `PlanExecutionRecordRepository` |
| `Entity` | JPA 实体类 | `PlanExecutionRecordEntity` |
| `VO` | 值对象 | `PlanExecutionRecord` |
| `PO` | 持久化对象 | `PlanTemplateVersion` |
| `DTO` | 数据传输对象 | `McpServerConfigDTO` |
| `Factory` | 工厂类 | `PlanningFactory`, `PlanExecutorFactory` |
| `Helper` | 辅助类 | `AgentInterruptionHelper` |
| `Manager` | 管理类 | `McpCacheManager` |
| `Provider` | 提供者 | `ToolCallbackProvider` |
| `Executor` | 执行器 | `DynamicToolPlanExecutor` |
| `Config` | 配置类 | `LynxeProperties` |
| `Exception` | 异常类 | `PlanException`, `ParameterValidationException` |
| `Enum` | 枚举类 | `ExecutionStatus`, `PlanTemplateAccessLevel` |
| `Interface` | 接口类 | `IMcpService`, `PlanningToolInterface` |

#### 示例
```java
// ✅ 正确
public class PlanTemplateService { }
public interface IMcpService { }
public class DynamicToolPlanExecutor extends AbstractPlanExecutor { }
public class PlanTemplateConfigException extends RuntimeException { }
public class ExecutionStatus { } // 枚举

// ❌ 错误
public class planTemplateService { }
public class service { }
public class DoSomething { }
public class MyException extends Exception { }
```

### 接口命名 (Interface Naming)

#### 规则
- **PascalCase**
- 可以使用 `I` 前缀（项目采用此风格）
- 描述能力的名词或形容词

#### 示例
```java
// ✅ 正确
public interface IMcpService { }
public interface PlanningToolInterface { }
public interface ToolCallbackProvider { }

// ❌ 错误
public interface mcpService { }
public interface Service { }
```

### 方法命名 (Method Naming)

#### 规则
- **camelCase**（首字母小写）
- 使用动词开头
- 返回 boolean 的方法使用 `is`, `has`, `can` 等前缀

#### 方法前缀约定

| 前缀 | 用途 | 示例 |
|------|------|------|
| `get` | 获取数据 | `getDetails()`, `getLatestVersion()` |
| `set` | 设置数据 | `setProperties()`, `setConfiguration()` |
| `is` | 判断状态（返回 boolean） | `isCompleted()`, `isRunning()` |
| `has` | 判断拥有（返回 boolean） | `hasPermission()`, `hasSubPlans()` |
| `can` | 判断能力（返回 boolean） | `canExecute()`, `canInterrupt()` |
| `create` | 创建对象 | `createPlan()`, `createExecutor()` |
| `save` | 保存数据 | `saveMcpServers()`, `saveToVersionHistory()` |
| `update` | 更新数据 | `updatePlanTemplate()`, `updateConfig()` |
| `delete` | 删除数据 | `deleteById()`, `removeMcpServer()` |
| `find` | 查询数据 | `findById()`, `findMaxVersionIndex()` |
| `execute` | 执行操作 | `executePlan()`, `executeTool()` |
| `process` | 处理数据 | `processToolCalls()`, `processRequest()` |
| `build` | 构建对象 | `buildMessages()`, `buildSchema()` |
| `parse` | 解析数据 | `parsePlanJson()`, `parseToolResponse()` |
| `validate` | 验证数据 | `validateParameters()`, `validateConfig()` |
| `init` | 初始化 | `initialize()`, `initAgent()` |
| `start` | 启动 | `startExecution()`, `startServer()` |
| `stop` | 停止 | `stopExecution()`, `stopServer()` |
| `pause` | 暂停 | `pauseExecution()` |
| `resume` | 恢复 | `resumeExecution()` |
| `interrupt` | 中断 | `interruptAgent()` |
| `clear` | 清除 | `clearCache()`, `clearHistory()` |

#### 示例
```java
// ✅ 正确
public PlanExecutionRecord getDetails(String planId) { }
public boolean isCompleted() { }
public String createPlan(PlanTemplate template) { }
public void saveMcpServers(List<McpServerConfig> servers) { }
public boolean validateParameters(Map<String, Object> params) { }

// ❌ 错误
public PlanExecutionRecord Details(String planId) { }
public boolean completed() { }
public String plan(PlanTemplate template) { }
public void servers(List<McpServerConfig> servers) { }
```

### 变量命名 (Variable Naming)

#### 规则
- **camelCase**
- 使用有意义的名称
- 避免单字符变量（除循环变量）
- 常量使用 **UPPER_SNAKE_CASE**

#### 示例
```java
// ✅ 正确
private String planId;
private List<AgentExecutionRecord> agentExecutionSequence;
private static final int MAX_RETRIES = 3;
private static final String DEFAULT_SERVICE_GROUP = "default";

// ❌ 错误
private String pid;
private List<AgentExecutionRecord> list;
private static final int max = 3;
private String PlanId; // 首字母不应大写
```

### 常量命名 (Constant Naming)

#### 规则
- **UPPER_SNAKE_CASE**
- 使用 `final` 修饰
- 通常使用 `static`

#### 示例
```java
// ✅ 正确
public static final String DEFAULT_SERVICE_GROUP = "default";
public static final int MAX_EXECUTION_STEPS = 100;
private static final Logger LOGGER = LoggerFactory.getLogger(MyClass.class);

// ❌ 错误
public static final String defaultServiceGroup = "default";
public static final int max_steps = 100;
```

### 枚举命名 (Enum Naming)

#### 规则
- 枚举类名: **PascalCase**
- 枚举值: **UPPER_SNAKE_CASE**

#### 示例
```java
// ✅ 正确
public enum ExecutionStatus {
    PENDING,
    RUNNING,
    COMPLETED,
    FAILED,
    PAUSED
}

public enum PlanTemplateAccessLevel {
    READ_ONLY,
    EDITABLE,
    PRIVATE
}

// ❌ 错误
public enum executionStatus {
    pending,
    running,
    completed
}

public enum ExecutionStatus {
    Pending,  // 不应使用 PascalCase
    Running,
    Completed
}
```

### 泛型命名 (Generic Naming)

#### 规则
- 使用单个大写字母
- 常用约定:
  - `T`: Type
  - `E`: Element
  - `K`: Key
  - `V`: Value
  - `R`: Result

#### 示例
```java
// ✅ 正确
public class Repository<T> { }
public interface Function<K, V> { }
public class Service<R> { }

// ❌ 错误
public class Repository<Type> { }
public class Repository<t> { }
```

## TypeScript 命名规范

### 接口和类型命名 (Interface/Type Naming)

#### 规则
- 接口: **PascalCase**
- 类型别名: **PascalCase**
- 接口名不以 `I` 开头（与 Java 约定不同）

#### 示例
```typescript
// ✅ 正确
interface PlanExecutionRecord {
  currentPlanId: string;
  agentExecutionSequence: AgentExecutionRecord[];
}

type ExecutionStatus = 'pending' | 'running' | 'completed' | 'failed';

interface AgentExecutionRecord {
  agentName: string;
  status: ExecutionStatus;
}

// ❌ 错误
interface planExecutionRecord { }
interface IPlanExecutionRecord { }  // 不使用 I 前缀
type executionstatus = 'pending' | 'running';
```

### 函数命名 (Function Naming)

#### 规则
- **camelCase**
- 使用动词开头
- 组合式函数以 `use` 开头

#### 示例
```typescript
// ✅ 正确
function getPlanDetails(planId: string): Promise<PlanExecutionRecord> { }
function isExecutionCompleted(status: ExecutionStatus): boolean { }

// Composable
export function usePlanExecution() { }
export function useConversation() { }
export function useTool() { }

// ❌ 错误
function GetPlanDetails(planId: string) { }
function usePlan() { }  // 非组合式函数不应使用 use 前缀
```

### 变量和属性命名 (Variable/Property Naming)

#### 规则
- **camelCase**
- 使用有意义的名称
- 布局值使用 `is`, `has`, `can` 前缀

#### 示例
```typescript
// ✅ 正确
const planId: string = '';
const agentExecutionSequence: AgentExecutionRecord[] = [];
const isCompleted: boolean = false;
const hasSubPlans: boolean = false;

// ❌ 错误
const PlanId: string = '';
const list: AgentExecutionRecord[] = [];
const completed: boolean = false;  // 应该用 isCompleted
```

### 组件命名 (Component Naming)

#### 规则
- **PascalCase**
- 使用多单词组合
- 使用 `.vue` 扩展名

#### 示例
```typescript
// ✅ 正确
// AgentExecutionView.vue
export default {
  name: 'AgentExecutionView'
}

// ChatInterface.vue
export default {
  name: 'ChatInterface'
}

// ❌ 错误
// agentExecutionView.vue
export default {
  name: 'agentExecutionView'
}
```

### Store 命名 (Store Naming)

#### 规则
- **camelCase**
- 使用 `use` 作为前缀（组合式 API）
- 或者直接使用名词（Pinia store 定义）

#### 示例
```typescript
// ✅ 正确
// stores/planExecution.ts
export const usePlanExecutionStore = defineStore('planExecution', () => {
  // ...
})

// stores/conversation.ts
export const useConversationStore = defineStore('conversation', () => {
  // ...
})

// 或者直接命名
export const useAvailableTools = defineStore('availableTools', () => {
  // ...
})
```

## 数据库命名规范

### 表命名 (Table Naming)

#### 规则
- **snake_case**
- 使用复数形式
- 使用前缀区分模块（可选）

#### 示例
```sql
-- ✅ 正确
CREATE TABLE plan_execution_records (
    id BIGINT PRIMARY KEY,
    current_plan_id VARCHAR(255) NOT NULL
);

CREATE TABLE agent_execution_records (
    id BIGINT PRIMARY KEY,
    plan_id VARCHAR(255) NOT NULL
);

CREATE TABLE plan_template_versions (
    id BIGINT PRIMARY KEY,
    plan_template_id VARCHAR(255) NOT NULL
);

-- ❌ 错误
CREATE TABLE PlanExecutionRecord (  -- 不应使用 PascalCase
    id BIGINT PRIMARY KEY
);

CREATE TABLE plan_execution_record (  -- 应该用复数
    id BIGINT PRIMARY KEY
);
```

### 列命名 (Column Naming)

#### 规则
- **snake_case**
- 使用有意义的名称
- 主键: `id`
- 外键: `{table}_id`
- 时间戳: `{action}_time` 或 `{action}_at`

#### 示例
```sql
-- ✅ 正确
CREATE TABLE plan_execution_records (
    id BIGINT PRIMARY KEY,
    current_plan_id VARCHAR(255) NOT NULL,
    root_plan_id VARCHAR(255),
    parent_plan_id VARCHAR(255),
    user_request TEXT,
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- ❌ 错误
CREATE TABLE plan_execution_records (
    id BIGINT PRIMARY KEY,
    currentPlanId VARCHAR(255),  -- 不应使用 camelCase
    startTime TIMESTAMP,         -- 不应使用 camelCase
    time TIMESTAMP               -- 不够明确
);
```

### 索引命名 (Index Naming)

#### 规则
- `idx_{table}_{column}` - 普通索引
- `uk_{table}_{column}` - 唯一索引
- `fk_{table}_{column}` - 外键索引

#### 示例
```sql
-- ✅ 正确
CREATE INDEX idx_plan_execution_records_current_plan_id ON plan_execution_records(current_plan_id);
CREATE UNIQUE INDEX uk_mcp_configs_server_name ON mcp_configs(server_name);

-- ❌ 错误
CREATE INDEX plan_plan_id ON plan_execution_records(current_plan_id);
CREATE INDEX idx1 ON plan_execution_records(current_plan_id);
```

## API 命名规范

### RESTful API 路径命名

#### 规则
- **kebab-case**
- 使用名词复数
- 使用层级结构表达关系

#### 示例
```java
// ✅ 正确
GET    /api/executor/details/{planId}
POST   /api/executor/executeByToolNameAsync
GET    /api/plan-template/list
POST   /api/plan-template/generate
POST   /api/plan-template/save
GET    /api/config/group/{groupName}
POST   /api/config/batch-update
POST   /api/file-upload/upload

// ❌ 错误
GET    /api/executor/getDetails/{planId}        // 不应包含动词
POST   /api/executor/execute-by-tool-name-async  // 过于冗长
GET    /api/planTemplate/list                    // 不应使用 camelCase
GET    /api/plan_templates                       // 不应使用下划线
```

### 查询参数命名 (Query Parameter Naming)

#### 规则
- **camelCase**
- 使用有意义的名称

#### 示例
```java
// ✅ 正确
GET /api/plan-template/list?serviceGroup=research&accessLevel=readOnly
GET /api/executor/details?includeSubPlans=true&maxDepth=3

// ❌ 错误
GET /api/plan-template/list?service_group=research
GET /api/executor/details?include_sub_plans=true
```

## 文件和目录命名规范

### 文件命名

#### 规则
- **kebab-case** (推荐) 或 **PascalCase** (类文件)
- 使用有意义的名称
- 避免使用特殊字符

#### 示例
```
✅ 正确
plan-execution-record.ts
AgentExecutionView.vue
PlanTemplateService.java
application-mysql.yml
README.md

❌ 错误
planExecutionRecord.ts
agent-execution-view.vue
plan template service.java
application.mysql.yml
readme.md
```

### 目录命名

#### 规则
- **kebab-case** (前端) 或 **camelCase** (Java 包)
- 使用有意义的名称

#### 示例
```
✅ 正确
ui-vue3/src/components/
ui-vue3/src/composables/
ui-vue3/src/stores/
src/main/java/com/alibaba/cloud/ai/lynxe/

❌ 错误
ui-vue3/src/Components/
ui-vue3/src/Composables/
ui-vue3/src/component/
```

## 特殊命名约定

### 业务术语

| 中文 | 英文 | 缩写 | 说明 |
|------|------|------|------|
| 计划 | Plan | - | 执行计划 |
| 模板 | Template | - | 计划模板 |
| 代理 | Agent | - | AI 代理 |
| 工具 | Tool | - | 执行工具 |
| 执行 | Execution | Exec | 执行过程 |
| 记录 | Record | Rec | 执行记录 |
| 对话 | Conversation | Conv | 用户对话 |
| 消息 | Message | Msg | 聊天消息 |
| 步骤 | Step | - | 执行步骤 |
| 参数 | Parameter | Param | 输入参数 |
| 配置 | Configuration | Config | 系统配置 |
| 服务 | Service | Svc | 业务服务 |
| 仓库 | Repository | Repo | 数据仓库 |

### 状态命名

| 状态 | 英文 | 值 |
|------|------|-----|
| 待处理 | PENDING | `"pending"` |
| 运行中 | RUNNING | `"running"` |
| 已完成 | COMPLETED | `"completed"` |
| 失败 | FAILED | `"failed"` |
| 暂停 | PAUSED | `"paused"` |
| 空闲 | IDLE | `"idle"` |
| 完成 | FINISHED | `"finished"` |

### 访问级别命名

| 级别 | 英文 | 值 |
|------|------|-----|
| 只读 | READ_ONLY | `"readOnly"` |
| 可编辑 | EDITABLE | `"editable"` |
| 私有 | PRIVATE | `"private"` |
| 已发布 | PUBLISHED | `"published"` |
