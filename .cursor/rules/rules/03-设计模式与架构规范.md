## 架构分层模式

### 三层架构

Lynxe 采用经典的三层架构设计：

```
┌─────────────────────────────────────┐
│      Presentation Layer             │  表现层
│  (Controller + Frontend)            │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼───────────────────┐
│       Business Layer                │  业务层
│  (Service + Domain Logic)           │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼───────────────────┐
│      Data Access Layer              │  数据访问层
│  (Repository + Entity)              │
└─────────────────────────────────────┘
```

### 层次职责

#### 表现层 (Presentation Layer)
**位置**: `runtime/controller/`, 前端 `views/`, `components/`

**职责**:
- 接收用户请求
- 参数验证
- 返回响应
- 不包含业务逻辑

**示例**:
```java
// LynxeController.java
@RestController
@RequestMapping("/api/executor")
public class LynxeController {

    @PostMapping("/executeByToolNameAsync")
    public ResponseEntity<AsyncExecutionResponse> executeAsync(
            @RequestBody ExecuteByToolNameRequest request) {
        // 仅处理请求响应，业务逻辑委托给 Service
        AsyncExecutionResponse response = planningCoordinator.executeAsync(request);
        return ResponseEntity.ok(response);
    }
}
```

#### 业务层 (Business Layer)
**位置**: `*/service/`, `agent/`, `tool/`

**职责**:
- 实现业务逻辑
- 协调各个组件
- 事务管理
- 业务规则验证

**示例**:
```java
// PlanningCoordinator.java
@Service
public class PlanningCoordinator {

    public AsyncExecutionResponse executeAsync(ExecuteByToolNameRequest request) {
        // 1. 参数验证
        validateRequest(request);

        // 2. 创建执行计划
        AbstractExecutionPlan plan = createExecutionPlan(request);

        // 3. 执行计划
        String planId = planExecutor.execute(plan);

        // 4. 返回结果
        return buildResponse(planId);
    }
}
```

#### 数据访问层 (Data Access Layer)
**位置**: `*/repository/`

**职责**:
- 数据库操作
- 数据持久化
- 数据查询

**示例**:
```java
// PlanExecutionRecordRepository.java
@Repository
public interface PlanExecutionRecordRepository extends JpaRepository<PlanExecutionRecordEntity, Long> {

    Optional<PlanExecutionRecordEntity> findByCurrentPlanId(String planId);

    void deleteByCurrentPlanId(String planId);
}
```

### 层次调用规则

1. **上层可以调用下层**
   - Controller → Service → Repository

2. **下层不能调用上层**
   - Repository 不能调用 Service
   - Service 不能调用 Controller

3. **同层之间通过接口调用**
   - Service 之间通过接口交互
   - 避免直接依赖具体实现

## 核心设计模式

### 1. 工厂模式 (Factory Pattern)

**用途**: 创建对象，封装创建逻辑

**示例位置**:
- `PlanningFactory.java` - 创建计划模板
- `PlanExecutorFactory.java` - 创建执行器

```java
// PlanningFactory.java
@Component
public class PlanningFactory {

    public AbstractExecutionPlan createPlan(PlanTemplateConfigVO config) {
        switch (config.getPlanType()) {
            case "dynamic_agent":
                return new DynamicAgentExecutionPlan(config);
            case "react_agent":
                return new ReActAgentExecutionPlan(config);
            default:
                throw new IllegalArgumentException("Unknown plan type: " + config.getPlanType());
        }
    }
}
```

### 2. 策略模式 (Strategy Pattern)

**用途**: 定义算法族，可以互相替换

**示例位置**:
- `AbstractPlanExecutor` 及其子类
- 不同的 Agent 实现策略

```java
// 抽象执行器
public abstract class AbstractPlanExecutor {
    public abstract String execute(AbstractExecutionPlan plan);
}

// 动态工具执行策略
public class DynamicToolPlanExecutor extends AbstractPlanExecutor {
    @Override
    public String execute(AbstractExecutionPlan plan) {
        // 动态工具执行逻辑
    }
}

// ReAct 执行策略
public class ReActPlanExecutor extends AbstractPlanExecutor {
    @Override
    public String execute(AbstractExecutionPlan plan) {
        // ReAct 执行逻辑
    }
}
```

### 3. 模板方法模式 (Template Method Pattern)

**用途**: 定义算法骨架，子类实现具体步骤

**示例位置**:
- `BaseAgent.java` - Agent 执行模板
- `AbstractBaseTool.java` - 工具执行模板

```java
// BaseAgent.java
public abstract class BaseAgent {

    // 模板方法
    public final AgentExecutionResult execute(ExecutionContext context) {
        // 1. 初始化
        initialize(context);

        // 2. 准备执行
        prepareExecution(context);

        // 3. 执行（子类实现）
        AgentExecutionResult result = doExecute(context);

        // 4. 清理
        cleanup(context);

        return result;
    }

    // 抽象方法，子类实现
    protected abstract AgentExecutionResult doExecute(ExecutionContext context);

    // 钩子方法
    protected void initialize(ExecutionContext context) { }
    protected void prepareExecution(ExecutionContext context) { }
    protected void cleanup(ExecutionContext context) { }
}
```

### 4. 观察者模式 (Observer Pattern)

**用途**: 一对多依赖，当对象状态改变时通知依赖者

**示例位置**:
- `LynxeEventPublisher.java` - 事件发布
- `PlanExceptionEvent` - 计划异常事件

```java
// 事件发布者
@Component
public class LynxeEventPublisher {

    private final ApplicationEventPublisher eventPublisher;

    public void publishPlanException(PlanExceptionEvent event) {
        eventPublisher.publishEvent(event);
    }
}

// 事件监听者
@Component
public class PlanExceptionListener {

    @EventListener
    public void handlePlanException(PlanExceptionEvent event) {
        // 处理计划异常
        log.error("Plan exception occurred: {}", event.getMessage());
    }
}
```

### 5. 适配器模式 (Adapter Pattern)

**用途**: 将一个接口转换成另一个接口

**示例位置**:
- `OpenAIAdapterService.java` - OpenAI API 适配
- `McpService.java` - MCP 协议适配

```java
// OpenAI 适配器
@Service
public class OpenAIAdapterService {

    public LynxeResponse adaptFromOpenAI(OpenAIResponse openAIResponse) {
        // 将 OpenAI 响应转换为 Lynxe 响应格式
        LynxeResponse response = new LynxeResponse();
        // ... 转换逻辑
        return response;
    }
}
```

### 6. 构建者模式 (Builder Pattern)

**用途**: 分步骤构建复杂对象

**示例位置**:
- 各种复杂对象的构建

```java
// 执行上下文构建者
public class ExecutionContextBuilder {

    private String planId;
    private Map<String, Object> parameters;
    private List<String> uploadedFiles;

    public ExecutionContextBuilder planId(String planId) {
        this.planId = planId;
        return this;
    }

    public ExecutionContextBuilder parameters(Map<String, Object> parameters) {
        this.parameters = parameters;
        return this;
    }

    public ExecutionContextBuilder uploadedFiles(List<String> uploadedFiles) {
        this.uploadedFiles = uploadedFiles;
        return this;
    }

    public ExecutionContext build() {
        return new ExecutionContext(planId, parameters, uploadedFiles);
    }
}

// 使用
ExecutionContext context = new ExecutionContextBuilder()
    .planId("plan-123")
    .parameters(params)
    .uploadedFiles(files)
    .build();
```

### 7. 单例模式 (Singleton Pattern)

**用途**: 确保一个类只有一个实例

**示例位置**:
- Spring Bean 默认是单例
- 配置管理类

```java
// Spring 单例 Bean
@Service
public class McpCacheManager {

    private final Map<String, McpServerInfo> cache = new ConcurrentHashMap<>();

    // Spring 保证单例
    public void addToCache(String key, McpServerInfo info) {
        cache.put(key, info);
    }
}
```

### 8. 责任链模式 (Chain of Responsibility Pattern)

**用途**: 将请求沿着处理链传递

**示例位置**:
- 工具调用的处理链
- 参数验证链

```java
// 工具调用处理器
public interface ToolCallHandler {
    void handle(ToolCallContext context, ToolCallHandler next);
}

// 参数验证处理器
@Component
public class ParameterValidationHandler implements ToolCallHandler {

    @Override
    public void handle(ToolCallContext context, ToolCallHandler next) {
        // 1. 验证参数
        validateParameters(context);

        // 2. 传递给下一个处理器
        next.handle(context, next);
    }
}

// 权限检查处理器
@Component
public class PermissionCheckHandler implements ToolCallHandler {

    @Override
    public void handle(ToolCallContext context, ToolCallHandler next) {
        // 1. 检查权限
        checkPermission(context);

        // 2. 传递给下一个处理器
        next.handle(context, next);
    }
}
```

## 业务架构模式

### 1. Agent-Tool 模式

**描述**: Agent 作为决策者，Tool 作为执行者

**架构**:
```
User Request
    ↓
Agent (决策、规划)
    ↓
Tool (执行具体操作)
    ↓
Result
```

**示例代码**:
```java
// Agent 决策
@Component
public class ConfigurableDynaAgent extends BaseAgent {

    @Override
    protected AgentExecutionResult doExecute(ExecutionContext context) {
        // 1. 分析用户请求
        String analysis = analyzeRequest(context.getUserRequest());

        // 2. 选择合适的工具
        List<Tool> selectedTools = selectTools(analysis);

        // 3. 调用工具执行
        List<ToolResult> results = executeTools(selectedTools, context);

        // 4. 综合结果
        return synthesizeResults(results);
    }
}
```

### 2. Plan-Execute 模式

**描述**: 先创建执行计划，再按计划执行

**架构**:
```
User Request
    ↓
Plan Generation (创建计划)
    ↓
Plan Execution (执行计划)
    ↓
Record & Monitor (记录和监控)
```

**关键组件**:
- `PlanningCoordinator` - 计划协调器
- `PlanExecutor` - 计划执行器
- `PlanExecutionRecorder` - 执行记录器

**示例代码**:
```java
// 计划协调
@Service
public class PlanningCoordinator {

    public String executeByToolName(ExecuteByToolNameRequest request) {
        // 1. 加载计划模板
        PlanTemplate template = loadTemplate(request.getToolName());

        // 2. 创建执行计划
        AbstractExecutionPlan plan = planningFactory.createPlan(template);

        // 3. 执行计划
        String planId = planExecutor.execute(plan);

        // 4. 开始记录
        planExecutionRecorder.startRecording(planId);

        return planId;
    }
}
```

**描述**: 封装数据访问逻辑

**对象类型**:

- **VO** (Value Object): 值对象，业务数据传输
- **DTO** (Data Transfer Object): 数据传输对象，跨层传输

**示例**:
```java

public class PlanExecutionRecord {
    private String currentPlanId;
    private String userRequest;
    private List<AgentExecutionRecord> agentExecutionSequence;

public class PlanExecutionRecordDTO {

    // ... getter/setter
}

// 转换器
@Component
public class PlanExecutionRecordConverter {

    public PlanExecutionRecordEntity toEntity(PlanExecutionRecord vo) {
        // PO ← VO 转换
    }

    public PlanExecutionRecord toVO(PlanExecutionRecordEntity entity) {
        // VO ← PO 转换
    }
}
```

## 并发和异步模式

### 1. 异步执行模式

**描述**: 任务提交后立即返回，后台异步执行

**示例**:
```java
// 异步执行接口
@PostMapping("/executeByToolNameAsync")
public ResponseEntity<AsyncExecutionResponse> executeAsync(
        @RequestBody ExecuteByToolNameRequest request) {

    // 立即返回 planId
    String planId = planningCoordinator.submitForExecution(request);

    AsyncExecutionResponse response = new AsyncExecutionResponse();
    response.setPlanId(planId);
    response.setStatus("processing");

    return ResponseEntity.ok(response);
}

// 后台执行
@Async
public void executeInBackground(String planId, AbstractExecutionPlan plan) {
    try {
        planExecutor.execute(plan);
    } catch (Exception e) {
        handleExecutionError(planId, e);
    }
}
```

### 2. 线程池模式

**描述**: 使用线程池管理并发执行

**示例位置**:
- `LevelBasedExecutorPool.java` - 分层执行池
- `ParallelExecutionService.java` - 并行执行服务

```java
// 分层执行池
@Component
public class LevelBasedExecutorPool {

    private final ExecutorService highPriorityPool;
    private final ExecutorService normalPriorityPool;
    private final ExecutorService lowPriorityPool;

    public LevelBasedExecutorPool() {
        highPriorityPool = Executors.newFixedThreadPool(5);
        normalPriorityPool = Executors.newFixedThreadPool(10);
        lowPriorityPool = Executors.newFixedThreadPool(20);
    }

    public Future<?> submit(Runnable task, Priority priority) {
        switch (priority) {
            case HIGH:
                return highPriorityPool.submit(task);
            case NORMAL:
                return normalPriorityPool.submit(task);
            case LOW:
                return lowPriorityPool.submit(task);
            default:
                throw new IllegalArgumentException("Unknown priority: " + priority);
        }
    }
}
```

### 3. 并行执行模式

**描述**: 多个任务并行执行，提高效率

**示例位置**:
- `mapreduce/ParallelExecutionService.java`

```java
// 并行执行服务
@Service
public class ParallelExecutionService {

    private final ExecutorService executorService;

    public List<ToolResult> executeParallel(List<ToolCall> toolCalls) {
        List<Callable<ToolResult>> tasks = toolCalls.stream()
            .map(toolCall -> (Callable<ToolResult>) () -> executeTool(toolCall))
            .collect(Collectors.toList());

        try {
            List<Future<ToolResult>> futures = executorService.invokeAll(tasks);
            return futures.stream()
                .map(Future::get)
                .collect(Collectors.toList());
        } catch (InterruptedException | ExecutionException e) {
            throw new ParallelExecutionException("Parallel execution failed", e);
        }
    }
}
```

## 扩展性设计模式

### 1. 插件模式 (Plugin Pattern)

**描述**: 动态加载和扩展功能

**示例位置**:
- MCP 工具集成
- 动态工具加载

```java
// 工具注册表
@Component
public class McpToolRegistry {

    private final Map<String, Tool> toolRegistry = new ConcurrentHashMap<>();

    public void registerTool(String toolName, Tool tool) {
        toolRegistry.put(toolName, tool);
    }

    public Tool getTool(String toolName) {
        return toolRegistry.get(toolName);
    }

    public List<String> getRegisteredToolNames() {
        return new ArrayList<>(toolRegistry.keySet());
    }
}

// 动态加载 MCP 工具
@Service
public class McpToolLoader {

    private final McpToolRegistry registry;

    @PostConstruct
    public void loadMcpTools() {
        List<McpServerConfig> servers = mcpService.getMcpServers();
        for (McpServerConfig server : servers) {
            List<Tool> tools = loadToolsFromServer(server);
            tools.forEach(tool -> registry.registerTool(tool.getName(), tool));
        }
    }
}
```

### 2. 装饰器模式 (Decorator Pattern)

**描述**: 动态添加功能

**示例位置**:
- Agent 能力扩展
- Tool 功能增强

```java
// 基础 Agent
public abstract class BaseAgent implements Agent {
    public abstract AgentExecutionResult execute(ExecutionContext context);
}

// 带缓存装饰的 Agent
public class CachedAgent extends BaseAgent {

    private final Agent delegate;
    private final Cache<String, AgentExecutionResult> cache;

    @Override
    public AgentExecutionResult execute(ExecutionContext context) {
        String cacheKey = generateCacheKey(context);

        AgentExecutionResult result = cache.get(cacheKey);
        if (result != null) {
            return result;
        }

        result = delegate.execute(context);
        cache.put(cacheKey, result);
        return result;
    }
}

// 带日志装饰的 Agent
public class LoggingAgent extends BaseAgent {

    private final Agent delegate;

    @Override
    public AgentExecutionResult execute(ExecutionContext context) {
        log.info("Agent execution started: {}", context.getPlanId());
        long startTime = System.currentTimeMillis();

        try {
            AgentExecutionResult result = delegate.execute(context);
            long duration = System.currentTimeMillis() - startTime;
            log.info("Agent execution completed in {}ms", duration);
            return result;
        } catch (Exception e) {
            log.error("Agent execution failed", e);
            throw e;
        }
    }
}
```

## 架构设计原则

### SOLID 原则

1. **单一职责原则 (SRP)**
   - 每个类只有一个改变的理由
   - 例如: `PlanTemplateService` 只负责计划模板管理

2. **开闭原则 (OCP)**
   - 对扩展开放，对修改关闭
   - 例如: 新增工具类型不需要修改现有代码

3. **里氏替换原则 (LSP)**
   - 子类可以替换父类
   - 例如: `DynamicToolPlanExecutor` 可以替换 `AbstractPlanExecutor`

4. **接口隔离原则 (ISP)**
   - 客户端不应依赖它不需要的接口
   - 例如: 工具接口按功能拆分

5. **依赖倒置原则 (DIP)**
   - 依赖抽象而非具体实现
   - 例如: Service 依赖 Repository 接口

### DRY 原则 (Don't Repeat Yourself)

- 提取公共逻辑
- 使用继承和组合
- 创建工具类

### KISS 原则 (Keep It Simple, Stupid)

- 保持设计简单
- 避免过度设计
- 优先选择简单方案
