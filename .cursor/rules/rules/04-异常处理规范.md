## 异常分类

### 按来源分类

#### 1. 业务异常 (Business Exception)
**特点**: 可预期的业务错误，通常由于用户输入或业务规则导致

**示例**:
- 参数验证失败
- 资源不存在
- 权限不足
- 业务规则冲突

**处理方式**: 捕获并转换为友好的错误消息

#### 2. 系统异常 (System Exception)
**特点**: 不可预期的系统错误

**示例**:
- 数据库连接失败
- 网络超时
- 内存溢出
- 配置错误

**处理方式**: 记录详细日志，返回通用错误消息

#### 3. 第三方服务异常 (Third-party Exception)
**特点**: 调用外部服务失败

**示例**:
- LLM API 调用失败
- MCP 服务连接失败
- 文件存储服务异常

**处理方式**: 记录日志，实现重试机制

### 按层级分类

```
┌─────────────────────────────────────┐
│      Controller Layer                │  ← 捕获所有异常，转换为 HTTP 响应
├─────────────────────────────────────┤
│      Service Layer                   │  ← 捕获并处理业务异常
├─────────────────────────────────────┤
│      Repository Layer                │  ← 捕获数据访问异常
├─────────────────────────────────────┤
│      External Service Layer          │  ← 捕获第三方服务异常
└─────────────────────────────────────┘
```

## 异常处理原则

### 1. 早期捕获原则
在异常发生的最早位置进行处理

```java
// ✅ 正确 - 早期验证
public void executePlan(String planId) {
    if (planId == null || planId.isEmpty()) {
        throw new ParameterValidationException("planId cannot be null or empty");
    }
    // 执行逻辑
}

// ❌ 错误 - 延迟到调用方处理
public void executePlan(String planId) {
    // 执行逻辑，可能在后期才抛出 NullPointerException
}
```

### 2. 异常转换原则
将底层异常转换为业务异常

```java
// ✅ 正确 - 转换为业务异常
try {
    PlanExecutionRecord record = repository.findByPlanId(planId)
        .orElseThrow(() -> new PlanNotFoundException(planId));
} catch (DataAccessException e) {
    throw new DataAccessException("Failed to access plan data", e);
}

// ❌ 错误 - 直接抛出底层异常
try {
    PlanExecutionRecord record = repository.findByPlanId(planId)
        .orElseThrow(() -> new NoSuchElementException());
} catch (DataAccessException e) {
    throw e;  // 直接抛出底层异常
}
```

### 3. 异常链保持原则
保留原始异常信息

```java
// ✅ 正确 - 保留异常链
try {
    executePlan(planId);
} catch (PlanExecutionException e) {
    throw new BusinessException("Plan execution failed", e);
}

// ❌ 错误 - 丢失原始异常
try {
    executePlan(planId);
} catch (PlanExecutionException e) {
    throw new BusinessException("Plan execution failed");
}
```

### 4. 具体异常优先原则
捕获具体异常而非泛型异常

```java
// ✅ 正确 - 捕获具体异常
try {
    fileService.writeFile(content);
} catch (IOException e) {
    handleIOException(e);
} catch (SecurityException e) {
    handleSecurityException(e);
}

// ❌ 错误 - 捕获泛型异常
try {
    fileService.writeFile(content);
} catch (Exception e) {
    handleException(e);
}
```

## 异常处理策略

### Controller 层异常处理

**位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/runtime/controller/`

使用 `@RestControllerAdvice` 全局处理异常

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    private static final Logger logger = LoggerFactory.getLogger(GlobalExceptionHandler.class);

    // 业务异常
    @ExceptionHandler(LynxeException.class)
    public ResponseEntity<ErrorResponse> handleLynxeException(LynxeException e) {
        logger.warn("Business exception occurred: {}", e.getMessage());

        ErrorResponse response = new ErrorResponse();
        response.setCode(e.getErrorCode() != null ? e.getErrorCode() : "BUSINESS_ERROR");
        response.setMessage(e.getMessage());
        response.setTimestamp(LocalDateTime.now());

        return ResponseEntity.badRequest().body(response);
    }

    // 参数验证异常
    @ExceptionHandler(ParameterValidationException.class)
    public ResponseEntity<ErrorResponse> handleParameterValidationException(ParameterValidationException e) {
        logger.warn("Parameter validation failed: {} = {}", e.getParameterName(), e.getParameterValue());

        ErrorResponse response = new ErrorResponse();
        response.setCode("INVALID_PARAMETER");
        response.setMessage(e.getMessage());
        response.setTimestamp(LocalDateTime.now());

        return ResponseEntity.badRequest().body(response);
    }

    // 资源不存在异常
    @ExceptionHandler(PlanNotFoundException.class)
    public ResponseEntity<ErrorResponse> handlePlanNotFoundException(PlanNotFoundException e) {
        logger.warn("Plan not found: {}", e.getPlanId());

        ErrorResponse response = new ErrorResponse();
        response.setCode("PLAN_NOT_FOUND");
        response.setMessage(e.getMessage());
        response.setTimestamp(LocalDateTime.now());

        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(response);
    }

    // 系统异常
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleException(Exception e) {
        logger.error("Unexpected error occurred", e);

        ErrorResponse response = new ErrorResponse();
        response.setCode("INTERNAL_SERVER_ERROR");
        response.setMessage("An unexpected error occurred. Please try again later.");
        response.setTimestamp(LocalDateTime.now());

        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(response);
    }
}

// 错误响应类
@Data
public class ErrorResponse {
    private String code;
    private String message;
    private LocalDateTime timestamp;
}
```

### Service 层异常处理

**位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/planning/service/`

```java
@Service
public class PlanTemplateService {

    private static final Logger logger = LoggerFactory.getLogger(PlanTemplateService.class);

    public PlanTemplate savePlanTemplate(PlanTemplate template) {
        try {
            // 业务逻辑
            validateTemplate(template);
            return repository.save(template);
        } catch (PlanTemplateConfigException e) {
            logger.error("Failed to save plan template: {}", e.getMessage());
            throw e;  // 重新抛出业务异常
        } catch (DataAccessException e) {
            logger.error("Data access error while saving plan template", e);
            throw new DataAccessException("Failed to save plan template due to data access error", e);
        }
    }

    private void validateTemplate(PlanTemplate template) {
        if (template == null) {
            throw new ParameterValidationException("template", "Template cannot be null");
        }
        if (template.getName() == null || template.getName().isEmpty()) {
            throw new ParameterValidationException("name", "Template name cannot be empty");
        }
    }
}
```

### Repository 层异常处理

```java
@Repository
public interface PlanExecutionRecordRepository extends JpaRepository<PlanExecutionRecordEntity, Long> {

    @Query("SELECT p FROM PlanExecutionRecordEntity p WHERE p.currentPlanId = :planId")
    Optional<PlanExecutionRecordEntity> findByCurrentPlanId(@Param("planId") String planId);

    // 使用 @Transactional 确保数据一致性
    @Transactional
    default void deleteByPlanIdSafely(String planId) {
        try {
            findByCurrentPlanId(planId).ifPresent(this::delete);
        } catch (DataAccessException e) {
            throw new DataAccessException("Failed to delete plan record: " + planId, e);
        }
    }
}
```

## 日志记录规范

### 日志级别使用

| 级别 | 用途 | 示例 |
|------|------|------|
| ERROR | 系统错误，需要立即处理 | 数据库连接失败、内存溢出 |
| WARN | 业务异常，需要关注 | 参数验证失败、资源不存在 |
| INFO | 关键业务流程 | 计划开始执行、用户登录 |
| DEBUG | 调试信息 | 方法入参、中间结果 |

### 日志格式规范

```java
// ✅ 正确 - 结构化日志
logger.error("Plan execution failed: planId={}, error={}", planId, error.getMessage());
logger.warn("Invalid parameter: name={}, value={}", paramName, paramValue);
logger.info("Plan execution started: planId={}, userId={}", planId, userId);

// ❌ 错误 - 字符串拼接
logger.error("Plan execution failed: " + planId + " error: " + error.getMessage());
```

### 异常日志记录

```java
// ✅ 正确 - 记录完整异常栈
try {
    executePlan(planId);
} catch (PlanExecutionException e) {
    logger.error("Plan execution failed: planId={}, error={}", planId, e.getMessage(), e);
    throw e;
}

// ❌ 错误 - 只记录消息
try {
    executePlan(planId);
} catch (PlanExecutionException e) {
    logger.error("Plan execution failed: {}", e.getMessage());
    throw e;
}

// ❌ 错误 - 不记录异常
try {
    executePlan(planId);
} catch (PlanExecutionException e) {
    throw e;
}
```

### 敏感信息处理

```java
// ✅ 正确 - 脱敏处理
logger.info("User login: userId={}, ip={}", userId, maskIp(ip));
logger.info("API call: token={}, endpoint={}", maskToken(token), endpoint);

// 脱敏工具类
public class SecurityLogger {

    public static String maskIp(String ip) {
        if (ip == null) return "null";
        String[] parts = ip.split("\\.");
        if (parts.length == 4) {
            return parts[0] + "." + parts[1] + ".*.*";
        }
        return "***";
    }

    public static String maskToken(String token) {
        if (token == null || token.length() < 10) return "***";
        return token.substring(0, 4) + "..." + token.substring(token.length() - 4);
    }
}

// ❌ 错误 - 记录敏感信息
logger.info("User login: userId={}, ip={}", userId, ip);  // 完整 IP
logger.info("API call: token={}, endpoint={}", token, endpoint);  // 完整 token
```

## 错误响应规范

### HTTP 状态码使用

| 状态码 | 含义 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | 请求成功 |
| 400 | 请求错误 | 参数验证失败、业务规则违反 |
| 404 | 资源不存在 | 查询的资源不存在 |
| 500 | 服务器错误 | 未预期的系统错误 |

### 错误码规范

格式: `{模块}_{错误类型}_{具体错误}`

```
PLAN_NOT_FOUND              - 计划不存在
PLAN_EXECUTION_FAILED       - 计划执行失败
INVALID_PARAMETER           - 参数无效
TOOL_EXECUTION_FAILED       - 工具执行失败
MCP_CONNECTION_FAILED       - MCP 连接失败
INTERNAL_SERVER_ERROR       - 内部服务器错误
```

### 错误响应格式

```json
{
  "code": "PLAN_NOT_FOUND",
  "message": "Plan not found: plan-123",
  "timestamp": "2026-03-09T10:30:00",
  "details": {
    "planId": "plan-123"
  }
}
```

## 前端异常处理

**位置**: `ui-vue3/src/api/`, `ui-vue3/src/composables/`

### API 错误处理

```typescript
// api/lynxe-service.ts
import axios from 'axios';

const apiClient = axios.create({
  baseURL: '/api',
  timeout: 30000
});

// 请求拦截器
apiClient.interceptors.request.use(
  (config) => {
    return config
  },
  (error) => {
    return Promise.reject(error)
  }
)

// 响应拦截器
apiClient.interceptors.response.use(
  (response) => {
    return response.data
  },
  (error) => {
    // 统一错误处理
    if (error.response) {
      // 服务器响应错误
      const { status, data } = error.response

      switch (status) {
        case 400:
          console.error('Bad Request:', data.message)
          break
        case 404:
          console.error('Not Found:', data.message)
          break
        case 500:
          console.error('Server Error:', data.message)
          break
        default:
          console.error('Error:', data.message)
      }

      return Promise.reject({
        code: data.code || 'UNKNOWN_ERROR',
        message: data.message || 'An error occurred',
        status
      })
    } else if (error.request) {
      // 请求无响应
      console.error('No Response:', error.message)
      return Promise.reject({
        code: 'NETWORK_ERROR',
        message: 'Network error. Please check your connection.',
        status: 0
      })
    } else {
      // 请求配置错误
      console.error('Request Error:', error.message)
      return Promise.reject({
        code: 'REQUEST_ERROR',
        message: error.message,
        status: -1
      })
    }
  }
)

export default apiClient
```

## 异常处理最佳实践

### 1. 使用 try-with-resources

```java
// ✅ 正确
try (FileReader reader = new FileReader(file)) {
    // 读取文件
} catch (IOException e) {
    logger.error("Failed to read file", e);
}

// ❌ 错误
FileReader reader = new FileReader(file);
try {
    // 读取文件
} catch (IOException e) {
    logger.error("Failed to read file", e);
} finally {
    reader.close();  // 可能抛出异常
}
```

### 2. 避免吞没异常

```java
// ✅ 正确
try {
    executePlan(planId);
} catch (PlanExecutionException e) {
    logger.error("Plan execution failed", e);
    throw e;  // 重新抛出
}

// ❌ 错误
try {
    executePlan(planId);
} catch (PlanExecutionException e) {
    logger.error("Plan execution failed", e);
    // 不做任何处理，吞没异常
}
```

### 3. 提供有用的错误信息

```java
// ✅ 正确
throw new ParameterValidationException("planId",
    "Plan ID cannot be null or empty, received: " + planId);

// ❌ 错误
throw new IllegalArgumentException("Invalid parameter");
throw new RuntimeException("Error occurred");
```

### 4. 使用具体的异常类型

```java
// ✅ 正确
throw new PlanNotFoundException(planId);
throw new ParameterValidationException("toolName", "Tool name cannot be empty");
throw new ToolExecutionException(toolName, "Tool execution timeout");

// ❌ 错误
throw new RuntimeException("Plan not found");
throw new IllegalArgumentException("Invalid parameter");
throw new Exception("Tool execution failed");
```

| 场景 | 异常类型 | 处理方式 |
