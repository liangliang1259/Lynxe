## 系统概念

### Lynxe (系统)

**定义**: Lynxe 是一个基于 Spring AI Alibaba 构建的 AI Agent 管理系统，提供多代理协作、计划执行和工具集成能力。

**核心特性**:
- 纯 Java 实现
- 支持 LLM 多模型
- 可扩展的工具系统
- MCP 协议集成
- HTTP 服务接口

**版本**: 4.10.7

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/`

### Func-Agent (函数代理)

**定义**: Func-Agent 是 Lynxe 的核心概念，代表一个可执行的功能单元，包含明确的步骤、工具配置和参数定义。

**特点**:
- 精确控制执行细节
- 支持参数化配置
- 可发布为工具或 HTTP 服务
- 支持版本管理

**使用场景**:
- 数据查询和处理
- 文件操作
- API 调用
- 自动化流程

**配置位置**: `src/main/resources/prompts/startup-plans/`

**示例**:
```json
{
  "title": "研究助手",
  "steps": [
    {
      "stepRequirement": "搜索相关资料",
      "agentName": "ConfigurableDynaAgent",
      "toolNames": ["browser", "search"]
    },
    {
      "stepRequirement": "整理研究结果",
      "agentName": "ConfigurableDynaAgent",
      "toolNames": ["write-file"]
    }
  ]
}
```

### Workspace (工作空间)

**定义**: Workspace 是执行计划时的独立工作目录，包含执行所需的文件和输出。

**目录结构**:
```
./workspace/{planId}/
├── files/           # 同步的输入文件
├── output/          # 执行输出
└── temp/            # 临时文件
```

**生命周期**:
- 创建: 计划开始执行时
- 使用: 执行过程中读写
- 清理: 执行完成后（可配置）

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/workspace/`

## 执行相关概念

### Plan (计划)

**定义**: Plan 代表一个完整的执行任务，包含一个或多个步骤，每个步骤由 Agent 执行。

**类型**:
- **动态代理计划** (DynamicAgentExecutionPlan): 使用动态代理执行
- **ReAct 计划** (ReActAgentExecutionPlan): 使用 ReAct 模式执行

**属性**:
```java
public class AbstractExecutionPlan {
    private String planId;           // 计划 ID
    private String planType;         // 计划类型
    private List<ExecutionStep> steps; // 执行步骤
    private ExecutionContext context; // 执行上下文
}
```

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/runtime/entity/vo/AbstractExecutionPlan.java`

### Execution (执行)

**定义**: Execution 是计划的运行过程，从开始到结束的完整生命周期。

**状态流转**:
```
PENDING → RUNNING → COMPLETED
                ↓
             FAILED
```

**相关概念**:
- **PlanId**: 执行计划的唯一标识符
- **RootPlanId**: 根计划 ID（用于子计划）
- **ParentPlanId**: 父计划 ID

**记录位置**: `PlanExecutionRecord`

### Agent (代理)

**定义**: Agent 是具有决策能力的 AI 实体，负责分析任务、选择工具并执行操作。

**类型**:
- **BaseAgent**: 基础代理抽象类
- **DynamicAgent**: 动态代理，可自主选择工具
- **ConfigurableDynaAgent**: 可配置动态代理
- **ReActAgent**: ReAct 模式代理

**职责**:
- 分析用户请求
- 选择合适的工具
- 调用 LLM 进行决策
- 执行工具调用
- 综合结果

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/agent/`

### Step (步骤)

**定义**: Step 是执行计划中的一个原子操作单元。

**属性**:
```java
public class ExecutionStep {
    private String stepRequirement;  // 步骤要求
    private String agentName;        // 使用的代理
    private List<String> toolNames;  // 可用工具列表
    private String terminateColumns; // 终止条件
}
```

**状态跟踪**:
- `currentStepIndex`: 当前执行步骤索引
- `completed`: 步骤是否完成

### PlanExecutionRecord (执行记录)

**定义**: 记录计划执行的完整信息，包括所有步骤、Agent 执行序列和最终结果。

**包含信息**:
- 基本信息: planId, title, userRequest
- 时间信息: startTime, endTime
- 状态信息: status, completed, currentStepIndex
- 执行序列: agentExecutionSequence
- 子计划: subPlanExecutionRecords

**用途**:
- 执行过程追踪
- 结果查询
- 问题排查
- 审计日志

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/recorder/entity/vo/PlanExecutionRecord.java`

### AgentExecutionRecord (代理执行记录)

**定义**: 记录单个 Agent 的执行过程。

**包含信息**:
```java
public class AgentExecutionRecord {
    private String stepId;              // 步骤 ID
    private String agentName;           // 代理名称
    private String agentRequest;        // 代理请求
    private String result;              // 执行结果
    private String status;              // 执行状态
    private List<ThinkActRecord> thinkActRecords; // 思考-行动记录
}
```

### ThinkActRecord (思考-行动记录)

**定义**: 记录 Agent 的思考和行动过程，用于理解决策逻辑。

**包含信息**:
- **ThinkInput**: 思考输入（包含上下文和工具调用结果）
- **ThinkOutput**: 思考输出（LLM 的决策）
- **ActToolInfo**: 行动工具信息（调用的工具和参数）
- **ActionResult**: 行动结果（工具执行结果）
- **字符统计**: inputCharCount, outputCharCount
- **时间统计**: thinkStartTime, thinkEndTime, actStartTime, actEndTime

**用途**:
- 理解 Agent 决策过程
- 调试和优化
- 审计和合规

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/recorder/entity/vo/ThinkActRecord.java`

## 模板相关概念

### PlanTemplate (计划模板)

**定义**: PlanTemplate 是可重用的执行计划定义，包含步骤配置、工具选择和参数定义。

**组成部分**:
```java
public class PlanTemplateConfigVO {
    private String planTemplateId;    // 模板 ID
    private String title;             // 标题
    private String planType;          // 计划类型
    private List<ExecutionStep> steps; // 执行步骤
    private ToolConfig toolConfig;    // 工具配置
    private PlanTemplateAccessLevel accessLevel; // 访问级别
}
```

**访问级别**:
- **READ_ONLY**: 只读，可以执行但不能修改
- **EDITABLE**: 可编辑
- **PRIVATE**: 私有，仅创建者可见

### Version (版本)

**定义**: 计划模板的版本历史，支持回滚和比较。

**版本管理**:
- 每次保存创建新版本
- 版本号递增
- 智能比较避免冗余版本

**版本索引**:
- `versionIndex`: 版本号，从 0 开始递增
- 每个模板有独立的版本序列

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/planning/model/po/PlanTemplateVersion.java`

### ServiceGroup (服务组)

**定义**: 用于组织和分类 Func-Agent 的逻辑分组。

**用途**:
- 工具分类管理
- 权限控制
- 服务发现

**示例**:
- `research`: 研究相关工具
- `database`: 数据库操作工具
- `file`: 文件操作工具

**默认组**: `default`

## 工具相关概念

### Tool (工具)

**定义**: Tool 是 Agent 可以调用的功能单元，实现特定的操作或服务。

**分类**:
- **内置工具**: 文件、数据库、浏览器、Office 等
- **MCP 工具**: 外部 MCP 服务提供的工具

**工具定义**:
```java
public interface Tool {
    String getName();           // 工具名称
    String getDescription();    // 工具描述
    ToolSchema getSchema();     // 工具 Schema（参数定义）
    ToolResult execute(ToolInput input); // 执行工具
}
```

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/tool/`

### ToolSchema (工具模式)

**定义**: 定义工具的输入参数结构。

**组成部分**:
- 参数名称
- 参数类型
- 是否必需
- 描述信息
- 默认值

**示例**:
```json
{
  "name": "read_file",
  "description": "Read a file from the filesystem",
  "inputSchema": {
    "type": "object",
    "properties": {
      "path": {
        "type": "string",
        "description": "Path to the file"
      }
    },
    "required": ["path"]
  }
}
```

### ToolResult (工具结果)

**定义**: 工具执行的返回结果。

**结构**:
```java
public class ToolResult {
    private boolean success;       // 是否成功
    private Object data;          // 返回数据
    private String error;         // 错误信息
    private Map<String, Object> metadata; // 元数据
}
```

### ToolRegistry (工具注册表)

**定义**: 管理所有可用工具的中心注册表。

**功能**:
- 工具注册和注销
- 工具查找
- 工具列表查询

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/tool/ToolRegistry.java`

## 协作相关概念

### Conversation (对话)

**定义**: 用户与系统之间的交互会话。

**组成部分**:
- **Dialog**: 对话框，包含一组消息
- **Message**: 消息，包含用户输入或系统响应

**相关服务**:
- `MemoryService`: 对话记忆管理
- `ConversationService`: 对话服务

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/workspace/conversation/`

### Memory (记忆)

**定义**: 系统保存的对话历史和上下文信息。

**用途**:
- 跨会话信息保留
- 上下文理解
- 个性化体验

**配置**:
```yaml
lynxe:
  general:
    enableConversationMemory: true
    maxConversationHistory: 100
```

### Namespace (命名空间)

**定义**: 用于隔离不同租户或用户的工作空间和资源。

**用途**:
- 多租户支持
- 资源隔离
- 权限管理

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/workspace/namespace/`

## MCP 相关概念

### MCP (Model Context Protocol)

**定义**: 一个开放协议，用于 AI 应用与外部数据源和工具之间的连接。

**官方文档**: https://modelcontextprotocol.io/

**在 Lynxe 中**:
- 支持作为 MCP 客户端
- 可连接到多个 MCP 服务器
- 动态加载 MCP 工具

### McpServer (MCP 服务器)

**定义**: 提供 MCP 接口的外部服务。

**连接类型**:
- **Process**: 通过进程间通信连接
- **HTTP**: 通过 HTTP 协议连接

**配置**:
```json
{
  "serverName": "filesystem-server",
  "connectionType": "process",
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path"]
}
```

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/mcp/`

## 配置相关概念

### ConfigEntity (配置实体)

**定义**: 系统配置的持久化实体。

**配置组**:
- **lynxe.agent**: Agent 相关配置
- **lynxe.general**: 通用配置
- **lynxe.browser**: 浏览器配置
- **lynxe.mcpServiceLoader**: MCP 配置

**配置更新**:
- 运行时更新
- 无需重启服务
- 立即生效

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/config/entity/ConfigEntity.java`

### LynxeProperties

**定义**: 核心配置属性类。

**注解**:
```java
@ConfigProperty(group = "lynxe", subGroup = "agent", key = "maxSteps")
private volatile Integer maxSteps = 100;
```

**代码位置**: `src/main/java/com/alibaba/cloud/ai/lynxe/config/LynxeProperties.java`
