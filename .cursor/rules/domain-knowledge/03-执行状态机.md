## 计划执行状态机

### 状态定义

| 状态 | 说明 | 代码值 | 可转换到 |
|------|------|--------|----------|
| PENDING | 待处理，计划已创建等待执行 | `pending` | RUNNING, FAILED |
| RUNNING | 运行中，正在执行 | `running` | COMPLETED, FAILED, PAUSED |
| COMPLETED | 已完成，执行成功 | `completed` | - |
| FAILED | 失败，执行出错 | `failed` | - |
| PAUSED | 暂停，执行被中断 | `paused` | RUNNING, FAILED |

### 状态转换图

```
     ┌──────────────┐
     │   PENDING    │  计划已创建
     └──────┬───────┘
            │
            │ 开始执行
            ▼
     ┌──────────────┐
     │   RUNNING    │  正在执行
     └──────┬───────┘
            │
     ┌──────┴──────┐
     │             │
     │ 完成        │ 失败/暂停
     ▼             ▼
┌─────────┐  ┌─────────┐  ┌─────────┐
│COMPLETED│  │ FAILED  │  │ PAUSED  │
└─────────┘  └─────────┘  └─────────┘
```

### 状态转换条件

#### PENDING → RUNNING

**触发条件**:
- 执行器开始执行计划

**代码位置**:
```java
// runtime/executor/DynamicToolPlanExecutor.java
public String execute(AbstractExecutionPlan plan) {
    String planId = plan.getPlanId();

    // 1. 开始记录
    recorder.startRecording(planId, context);
    recorder.updateStatus(planId, ExecutionStatus.PENDING);

    // 2. 开始执行，状态变为 RUNNING
    recorder.updateStatus(planId, ExecutionStatus.RUNNING);

    return planId;
}
```

#### RUNNING → COMPLETED

**触发条件**:
- 所有步骤执行完成
- 执行结果成功

**代码位置**:
```java
// recorder/service/PlanExecutionRecorder.java
public void markAsCompleted(String planId) {
    PlanExecutionRecord record = getRecord(planId);
    record.setCompleted(true);
    record.setStatus(ExecutionStatus.COMPLETED);
    record.setEndTime(LocalDateTime.now());
    updateRecord(record);
}
```

#### RUNNING → FAILED

**触发条件**:
- 执行过程中发生异常
- Agent 执行失败
- 工具调用失败

**代码位置**:
```java
// runtime/executor/DynamicToolPlanExecutor.java
try {
    agent.execute(context);
    recorder.markAsCompleted(planId);
} catch (Exception e) {
    recorder.markAsFailed(planId, e.getMessage());
    throw new PlanExecutionException(planId, "Execution failed", e);
}
```

#### RUNNING → PAUSED

**触发条件**:
- 用户主动中断
- 系统检测到需要用户输入

**代码位置**:
```java
// runtime/service/AgentInterruptionHelper.java
public void interrupt(String planId) {
    PlanExecutionRecord record = recorder.getRecord(planId);

    if (record.isRunning()) {
        record.setStatus(ExecutionStatus.PAUSED);
        recorder.updateRecord(record);

        // 中断 Agent 执行
        Agent agent = agentRegistry.getAgent(planId);
        agent.interrupt();
    }
}
```

#### PAUSED → RUNNING

**触发条件**:
- 用户恢复执行

**代码位置**:
```java
// runtime/service/AgentInterruptionHelper.java
public void resume(String planId) {
    PlanExecutionRecord record = recorder.getRecord(planId);

    if (record.getStatus() == ExecutionStatus.PAUSED) {
        record.setStatus(ExecutionStatus.RUNNING);
        recorder.updateRecord(record);

        // 恢复 Agent 执行
        Agent agent = agentRegistry.getAgent(planId);
        agent.resume();
    }
}
```

## Agent 执行状态机

### 状态定义

| 状态 | 说明 | 代码值 | 可转换到 |
|------|------|--------|----------|
| IDLE | 空闲，Agent 未运行 | `IDLE` | RUNNING |
| RUNNING | 运行中，Agent 正在执行 | `RUNNING` | FINISHED, IDLE |

### 状态转换图

```
┌──────────┐
│  IDLE    │  Agent 创建
└────┬─────┘
     │
     │ 开始执行
     ▼
┌──────────┐
│ RUNNING  │  正在执行
└────┬─────┘
     │
     │ 执行完成
     ▼
┌──────────┐
│ FINISHED │  执行完成
└──────────┘
```

### 状态转换实现

**代码位置**: `agent/BaseAgent.java`

```java
public abstract class BaseAgent {

    private volatile ExecutionStatus status = ExecutionStatus.IDLE;

    public final AgentExecutionResult execute(ExecutionContext context) {
        // 1. 设置状态为 RUNNING
        this.status = ExecutionStatus.RUNNING;

        try {
            // 2. 执行 Agent 逻辑
            AgentExecutionResult result = doExecute(context);

            // 3. 状态变为 FINISHED
            this.status = ExecutionStatus.FINISHED;

            return result;
        } catch (Exception e) {
            // 4. 错误时状态变为 IDLE
            this.status = ExecutionStatus.IDLE;
            throw e;
        }
    }

    public ExecutionStatus getStatus() {
        return status;
    }
}
```

## 工具调用状态机

### 状态定义

| 状态 | 说明 | 可转换到 |
|------|------|----------|
| PENDING | 待执行 | EXECUTING, FAILED |
| EXECUTING | 执行中 | SUCCESS, FAILED |
| SUCCESS | 成功 | - |
| FAILED | 失败 | PENDING (重试) |

### 状态转换图

```
┌──────────┐
│ PENDING  │  工具调用请求
└────┬─────┘
     │
     │ 开始执行
     ▼
┌──────────┐
│EXECUTING │  正在执行
└────┬─────┘
     │
     ├─────┐
     │     │
     ▼     ▼
┌─────┐ ┌─────┐
│SUCCESS│ │FAILED│
└─────┘ └─────┘
```

### 状态转换实现

**代码位置**: `tool/AbstractBaseTool.java`

```java
public abstract class AbstractBaseTool implements Tool {

    public final ToolResult execute(ToolInput input) {
        try {
            // 1. 参数验证
            validateParameters(input);

            // 2. 状态变为 EXECUTING
            setToolStatus(ToolStatus.EXECUTING);

            // 3. 执行工具逻辑
            Object result = doExecute(input);

            // 4. 状态变为 SUCCESS
            setToolStatus(ToolStatus.SUCCESS);

            return ToolResult.success(result);

        } catch (Exception e) {
            // 5. 错误时状态变为 FAILED
            setToolStatus(ToolStatus.FAILED);
            logger.error("Tool execution failed", e);

            return ToolResult.failure(e.getMessage());
        }
    }

    protected abstract Object doExecute(ToolInput input);
}
```

## 状态转换规则

### 允许的转换

| 从状态 | 到状态 | 触发条件 |
|--------|--------|----------|
| PENDING | RUNNING | 执行开始 |
| PENDING | FAILED | 执行前验证失败 |
| RUNNING | COMPLETED | 所有步骤完成 |
| RUNNING | FAILED | 执行过程异常 |
| RUNNING | PAUSED | 用户中断 |
| PAUSED | RUNNING | 用户恢复 |
| PAUSED | FAILED | 恢复失败 |

### 禁止的转换

| 从状态 | 到状态 | 原因 |
|--------|--------|------|
| COMPLETED | RUNNING | 已完成不能重新执行 |
| FAILED | RUNNING | 失败后需创建新执行 |
| COMPLETED | FAILED | 已完成不能变为失败 |

### 状态验证

```java
// recorder/service/PlanExecutionRecorder.java
public void updateStatus(String planId, ExecutionStatus newStatus) {
    PlanExecutionRecord record = getRecord(planId);
    ExecutionStatus currentStatus = record.getStatus();

    // 验证状态转换是否合法
    if (!isValidTransition(currentStatus, newStatus)) {
        throw new IllegalStateException(
            String.format("Invalid status transition: %s -> %s",
                currentStatus, newStatus)
        );
    }

    record.setStatus(newStatus);
    updateRecord(record);
}

private boolean isValidTransition(ExecutionStatus from, ExecutionStatus to) {
    // 定义允许的状态转换
    Map<ExecutionStatus, Set<ExecutionStatus>> validTransitions = Map.of(
        ExecutionStatus.PENDING, Set.of(ExecutionStatus.RUNNING, ExecutionStatus.FAILED),
        ExecutionStatus.RUNNING, Set.of(ExecutionStatus.COMPLETED, ExecutionStatus.FAILED, ExecutionStatus.PAUSED),
        ExecutionStatus.PAUSED, Set.of(ExecutionStatus.RUNNING, ExecutionStatus.FAILED)
    );

    Set<ExecutionStatus> allowed = validTransitions.get(from);
    return allowed != null && allowed.contains(to);
}
```

## 错误处理与恢复

### 错误状态处理

#### 1. 计划执行失败

```
RUNNING → FAILED
     │
     ├─ 记录错误信息
     ├─ 清理资源
     ├─ 发布异常事件
     └─ 通知用户
```

**实现**:
```java
// runtime/service/PlanningCoordinator.java
@EventListener
public void handlePlanException(PlanExceptionEvent event) {
    String planId = event.getPlanId();

    // 1. 更新状态为 FAILED
    recorder.markAsFailed(planId, event.getMessage());

    // 2. 记录错误信息
    PlanExecutionRecord record = recorder.getRecord(planId);
    record.setErrorMessage(event.getMessage());
    recorder.updateRecord(record);

    // 3. 清理资源
    cleanupResources(planId);

    // 4. 发送通知
    sendFailureNotification(planId, event.getMessage());
}
```

#### 2. Agent 执行失败

```
RUNNING → FINISHED (但 result = null)
     │
     ├─ 记录错误信息
     ├─ 标记步骤失败
     └─ 继续下一步或终止计划
```

**实现**:
```java
// agent/BaseAgent.java
public AgentExecutionResult execute(ExecutionContext context) {
    try {
        return doExecute(context);
    } catch (Exception e) {
        // 构建失败结果
        AgentExecutionResult result = new AgentExecutionResult();
        result.setSuccess(false);
        result.setErrorMessage(e.getMessage());
        result.setStatus(ExecutionStatus.FINISHED);
        return result;
    }
}
```

#### 3. 工具调用失败

```
EXECUTING → FAILED
     │
     ├─ 记录失败信息
     ├─ 根据配置决定是否重试
     └─ 返回错误给 Agent
```

**实现**:
```java
// tool/AbstractBaseTool.java
private ToolResult executeWithRetry(ToolInput input, int maxRetries) {
    int retryCount = 0;

    while (retryCount < maxRetries) {
        try {
            return doExecute(input);
        } catch (Exception e) {
            retryCount++;
            if (retryCount >= maxRetries) {
                return ToolResult.failure("Tool execution failed after " +
                    maxRetries + " retries: " + e.getMessage());
            }

            // 等待后重试
            try {
                Thread.sleep(1000 * retryCount);
            } catch (InterruptedException ie) {
                Thread.currentThread().interrupt();
                return ToolResult.failure("Tool execution interrupted");
            }
        }
    }

    return ToolResult.failure("Max retries exceeded");
}
```

### 恢复策略

#### 1. 自动重试

**适用场景**:
- 网络超时
- 临时性错误
- 工具调用失败

**配置**:
```yaml
lynxe:
  agent:
    maxRetries: 3
    retryDelay: 1000  # 毫秒
```

#### 2. 用户介入

**适用场景**:
- 需要用户输入
- 需要用户确认
- 权限不足

**实现**:
```java
// recorder/entity/vo/PlanExecutionRecord.java
private UserInputWaitState userInputWaitState;

public void waitForUserInput(Map<String, Object> inputFields) {
    this.userInputWaitState = new UserInputWaitState();
    this.userInputWaitState.setInputFields(inputFields);
    this.userInputWaitState.setStatus("waiting");

    // 状态变为 PAUSED
    this.status = ExecutionStatus.PAUSED;
}

public void submitUserInput(Map<String, Object> userInput) {
    this.userInputWaitState.setUserInput(userInput);
    this.userInputWaitState.setStatus("submitted");

    // 状态变为 RUNNING
    this.status = ExecutionStatus.RUNNING;
}
```

#### 3. 从检查点恢复

**适用场景**:
- 长时间运行的任务
- 支持断点续传

**实现**:
```java
// runtime/executor/DynamicToolPlanExecutor.java
public void resumeFromCheckpoint(String planId, int stepIndex) {
    PlanExecutionRecord record = recorder.getRecord(planId);

    // 从指定步骤恢复
    record.setCurrentStepIndex(stepIndex);
    record.setStatus(ExecutionStatus.RUNNING);

    // 重新执行
    executeFromStep(record, stepIndex);
}
```

// ui-vue3/src/composables/usePlanExecution.ts
export function usePlanExecution() {
  const status = ref<ExecutionStatus>('pending')
  const progress = ref(0)

  const startPolling = async (planId: string) => {
    while (true) {
      const record = await getDetails(planId)

      // 更新状态
      status.value = record.status || 'pending'
      progress.value = calculateProgress(record)

      // 检查是否完成
      if (record.completed || record.status === 'failed') {
        break
      }

      await sleep(1000)  // 每秒轮询
    }
  }

  return { status, progress, startPolling }
}

### 状态恢复

```java
// runtime/service/PlanningCoordinator.java
public void restoreExecution(String planId) {
    // 1. 从数据库加载状态
    PlanExecutionRecord record = recorder.getRecord(planId);

    if (record == null) {
        throw new PlanNotFoundException(planId);
    }

    // 2. 检查状态是否可恢复
    if (record.isCompleted()) {
        throw new IllegalStateException("Plan already completed: " + planId);
    }

    // 3. 恢复执行
    if (record.getStatus() == ExecutionStatus.PAUSED) {
        resumeExecution(planId);
    } else if (record.getStatus() == ExecutionStatus.FAILED) {
        retryExecution(planId);
    }
}
```
