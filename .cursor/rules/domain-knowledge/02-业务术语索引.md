## 执行相关术语

### 执行状态

| 状态 | 说明 | 使用场景 |
|------|------|----------|
| PENDING | 待处理 | 计划已创建，等待执行 |
| RUNNING | 运行中 | 正在执行 |
| COMPLETED | 已完成 | 执行成功完成 |
| FAILED | 失败 | 执行失败 |
| PAUSED | 暂停 | 执行被暂停 |

### Agent 状态

| 状态 | 说明 | 位置 |
|------|------|------|
| IDLE | 空闲 | Agent 未运行 |
| RUNNING | 运行中 | Agent 正在执行 |
| FINISHED | 已完成 | Agent 执行完成 |

## 术语使用规范

### 命名约定

1. **英文优先**: 代码和文档中使用英文术语
2. **统一翻译**: 中文文档中使用统一的翻译
3. **保留原文**: 专业术语首次出现时保留英文

### 示例

```
✅ 正确:
- Agent（代理）负责决策和执行
- 使用 LLM（大语言模型）进行推理
- MCP（Model Context Protocol）集成

❌ 错误:
- 智能体负责决策和执行
- 使用大模型进行推理
- 模型上下文协议集成
```
